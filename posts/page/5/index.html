    Adding `.htaccess` to Apache Container `/tmp` Directory  2 min read   Apr 14, 2021
    <p>For some time now we’ve had a problem lurking in <a href="https://digital.grinnell.edu">Digital.Grinnell</a>, when large files are opened for viewing or download one of the <em>DG</em> services makes a temporary copy of the file in the <em>Apache</em> container’s <code>/tmp</code> directory. Locally, and in staging I’ve debugged the code that is responsible for removing the temporary file once the operation is complete. Running locally or in staging the process does its job, the temporary files get deleted soon after creation, but this never happens in production. The result, our root disk on the production server fills up after a few days of use, and the server stops serving content. Even more sinister, the server doesn’t crash and restart – a condition that would also clear the offending <code>/tmp</code> files – it doesn’t even report a fatal error, it just refuses to serve content, which is really its only function.
